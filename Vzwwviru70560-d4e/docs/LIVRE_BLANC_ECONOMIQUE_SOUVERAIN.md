## 1. VISION ET PRINCIPES FONDAMENTAUX

### 1.1 Le Problème de l'Ancien Système

Le système bancaire traditionnel souffre de défauts structurels :
- **Extraction** : Les frais et intérêts drainent la richesse vers le haut
- **Contrôle** : Les comptes peuvent être gelés arbitrairement
- **Inflation** : La création monétaire dilue l'épargne des citoyens
- **Opacité** : L'allocation des ressources est invisible

### 1.2 La Solution Souveraine

L'Arche propose un système économique **vivant** qui :
- **Redistribue** la valeur vers ceux qui la créent
- **Protège** les avoirs contre le contrôle externe
- **Stabilise** la valeur par des mécanismes automatiques
- **Transparence** totale via la blockchain Hedera

## 2. ARCHITECTURE MONÉTAIRE SOUVERAINE

### 2.1 L'Unité de Résonance (UR)

La monnaie souveraine de l'Arche, créée via le **Hedera Token Service (HTS)**.

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                         CARACTÉRISTIQUES DE L'UNITÉ DE RÉSONANCE               │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│   Nom du Token      : Unité de Résonance (UR)                                  │
│   Réseau            : Hedera Hashgraph (HTS)                                   │
│   Décimales         : 8 (comme le Bitcoin)                                     │
│   Émission Initiale : Limitée, basée sur les actifs de garantie               │
│   Gouvernance       : Contrôlée par le Souverain via smart contracts          │
│                                                                                 │
│   Vitesse           : ~10,000 TPS (transactions par seconde)                   │
│   Frais             : < 0.001 $ par transaction                                │
│   Finalité          : 3-5 secondes                                             │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘
```

### 2.2 Ancrage de Valeur

L'Unité de Résonance est garantie par un panier d'actifs :

| Type d'Actif | Proportion | Description |
|--------------|------------|-------------|
| **Réserve Fiat** | Variable | CAD/USD en compte de liquidité |
| **Services** | Variable | Puissance de calcul, accès API |
| **Actifs Tangibles** | Variable | Terrains, infrastructures |
| **Capital Humain** | Variable | Expertise des membres |

### 6.2 Calcul du Score de Résonance (pour éligibilité)

```
resonance_score =
    (activity_score * 0.25) +
    (contribution_score * 0.25) +
    (tenure_score * 0.20) +
    (investment_score * 0.15) +
    (referral_score * 0.15)

où:
  activity_score    = transactions_count / avg_transactions * 100
  contribution_score = contributions_value / avg_contribution * 100
  tenure_score       = months_active / 12 * 100 (plafonné à 100)
  investment_score   = ur_balance / avg_balance * 100
  referral_score     = active_referrals * 10 (plafonné à 100)
```

```

   - Solde suffisant

## 9. SCÉNARIOS DE CRISE ET RÉPONSES AUTOMATIQUES

### 9.1 Matrice de Réponse

| Scénario | Détection | Réponse Automatique | Action Souverain |
|----------|-----------|---------------------|------------------|
| **Inflation rapide** | Vélocité > 2x cible | ↑ frais, ↓ émissions | Communiquer |
| **Déflation** | Vélocité < 0.5x cible | ↓ frais, ↑ bonus | Stimuler |
| **Bank Run** | Conversions > 30%/jour | Bouclier de liquidité | Gérer file |
| **Attaque externe** | Transactions anormales | Pause + vérification | Enquêter |
| **Perte de confiance** | Désabonnements massifs | Mode survie | Rassurer |
| **Faillite fournisseur** | API down | Basculer backup | Trouver alternative |

## ANNEXE B : FORMULES CLÉS

```
Taux de conversion UR/CAD:
  ur_cad_rate = liquidity_pool.fiat_balance / ur_total_supply

Éligibilité ristourne:
  eligible = (resonance_score >= activity_threshold_score)
             AND (account_age_days >= 30)

Montant ristourne individuelle:
  rebate = surplus * resonance_rebate_pct * (user_score / sum_all_scores)

Ratio de réserve:
  reserve_ratio = liquidity_pool.fiat_balance / (ur_total_supply * ur_cad_rate)

Vélocité monétaire:
  velocity = total_transaction_volume_30d / average_money_supply_30d
