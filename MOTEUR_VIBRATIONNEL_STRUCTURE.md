```sql
-- forge_system.sql

1. BOARD DES ROLES (arche_roles)
   - Roles definis avec competences requises
   - min_resonance_score pour chaque role
   - Matching automatique membre <-> role

2. MATCHING ENGINE
   - calculate_role_match(member_id, role_id)
   - Score base sur: skills_match * 0.6 + resonance_match * 0.4
   - Suggestions automatiques de roles

3. AGENTS D'ETUDE DE MARCHE
   - oracle_financier: Surveille les marches
   - detecteur_foncier: Opportunites terrains
   - veille_tech: Brevets et innovations
   - analyse_concurrence: Veille concurrentielle

4. ESPACE PROJETS
   - forge_projects: Projets de construction
   - project_team_members: Equipes
   - project_disbursements: Flux financiers
```

### Cycle de Vie d'une Transition

```
1. DETECTION (Agents L4)
   - Agent detecte une opportunite/probleme
   - Genere un market_report

2. PROPOSITION
   - Si significatif: governance_proposal
   - Stake requis: proposal_minimum_ur_stake

3. DISCUSSION (3-7 jours)
   - Forum ouvert
   - Amendements possibles

4. VOTE
   - Pondere par resonance_score
   - Quorum: 33%, Approbation: 67%

5. EXECUTION
   - Si approuve: forge_project cree
   - Equipe assemblee via matching
   - Budget alloue

6. COMPLETION
   - Mesure de l'impact
   - Ajustement du Volant si necessaire
   - Mise a jour des resonance_scores
```
